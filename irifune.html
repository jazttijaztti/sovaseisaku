Gitとは ローカル上のファイルやコードのバージョンを管理する仕組み | コマンド | 説明 | | --- | --- | | `git -v` | Gitのバージョン確認 | | `git init` | Gitの管理を開始 | | `git add .` | ファイルを選択している | | `git commit -am "コメント"` | ステージした変更を履歴として記録する（コメントは履歴のメモ） | | `git branch` | ブランチの一覧を表示 | | `git branch ブランチ名` | 新しいブランチを作成 | | `git checkout ブランチ名` | ブランチを切り替え | | `git merge ブランチ名` | 他ブランチの変更を現在のブランチに取り込む | | `git log` | コミット履歴の確認 | | `tig` | Git履歴を視覚的に表示 | コンフリクトが起きた場合 mergeしたときに、重複したものがあると、conflictと表記され、重複している箇所が目立って表示される GitHubとは 複数人で共同作業を行う際に、リポジトリを共有・管理できるサービス | `git remote -v` | gitとgithubの接続を確認 GitHubに**鍵（公開鍵）を登録し、PCに鍵穴（秘密鍵）**を保存 | 操作 | コマンド | | 最新版を取得 | `git pull origin main` | | 変更をGitHubに送信 | `git push origin main` |
